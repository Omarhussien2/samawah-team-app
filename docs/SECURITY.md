# ملاحظات الأمان - سماوة

## مبادئ الأمان الأساسية

### 1. Row Level Security (RLS)
- ✅ مفعّل على جميع الجداول في Supabase
- ✅ Admin يرى كل شيء
- ✅ Project Manager يرى مشاريعه فقط
- ✅ Member يرى المشاريع التي هو عضو فيها فقط

### 2. Service Role Key
- ❌ لا تضع Service Role Key في الكود المرئي للمتصفح
- ✅ يُستخدم فقط في Server Components وAPI Routes الآمنة
- ✅ يُستخدم في Cron Job endpoints

### 3. Cron Secret
- ✅ جميع endpoints الـ Cron محمية بـ `CRON_SECRET`
- ✅ تأكد من قيمة عشوائية قوية (20+ حرف)
- ✅ احفظها فقط في Vercel Environment Variables

### 4. المفاتيح والأسرار
- ❌ لا ترفع `.env` أو `.env.local` على GitHub
- ✅ استخدم `.env.example` فقط كمرجع (بدون قيم حقيقية)
- ✅ جميع الأسرار في Vercel Environment Variables فقط
- ✅ جميع الأسرار في Supabase فقط

## أمان المصادقة

- يُستخدم Supabase Auth المبني على JWT
- الجلسات تنتهي تلقائياً
- لا يتم تخزين كلمات المرور في الكود
- Supabase تتعامل مع التشفير وإدارة الجلسات

---

## توصيات إضافية

### للبيئة الإنتاجية:
1. فعّل **Email Confirmation** في Supabase Auth للمستخدمين الجدد
2. فعّل **MFA (Multi-Factor Authentication)** في Supabase للمسؤولين
3. راقب **Supabase Logs** بانتظام
4. راجع **Vercel Access Logs** شهرياً

### لقاعدة البيانات:
1. لا تشارك كلمة مرور قاعدة البيانات
2. استخدم Supabase Backups (متاح في الخطة المدفوعة)
3. لا تنفّذ SQL مباشرةً في الإنتاج بدون نسخة احتياطية

### للبريد الإلكتروني:
1. استخدم App Password وليس كلمة المرور الأصلية
2. فعّل 2FA على حساب Gmail
3. لا تشارك SMTP credentials
